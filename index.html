Custom Image
커스텀 Dockerfile
공식 n8n 이미지 대신 Puppeteer와 여러 npm 패키지를 포함한 커스텀 이미지를 빌드합니다. 웹 스크래핑, 유튜브 파싱 등 확장 기능이 가능해집니다.
Dockerfile 
FROM node:20-alpine

RUN apk update && apk add --no-cache \
    chromium harfbuzz freetype ttf-freefont nss tini

ENV PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=true
ENV PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser

RUN npm install -g n8n@2.4.8 puppeteer iconv-lite axios crawlee cheerio youtubei

ENV NODE_FUNCTION_ALLOW_EXTERNAL=iconv-lite,axios,puppeteer,cheerio,youtubei
ENV NODE_FUNCTION_ALLOW_BUILTIN=*

ENTRYPOINT ["/sbin/tini", "--"]
CMD ["n8n"]
Backup Process
백업 스크립트 5단계
backup.sh는 서비스 중단 없이 실행됩니다. --no-executions 옵션으로 실행 기록을 제외할 수 있습니다.
1

n8n CLI Export

n8n 컨테이너 안에서 CLI를 사용해 워크플로우와 크리덴셜을 JSON 파일로 내보냅니다. 가장 안전한 백업 방식입니다.

$ docker exec n8n n8n export:workflow --all --separate --output=/tmp/workflows/ $ docker exec n8n n8n export:credentials --all --separate --output=/tmp/credentials/ $ docker cp n8n:/tmp/n8n_export/. ./backup/n8n_export/
2

PostgreSQL Dump

pg_dump로 전체 데이터베이스를 SQL 파일로 덤프합니다. --no-executions 옵션 사용 시 execution 테이블 데이터를 제외합니다.

$ docker exec postgres pg_dump -U root -d n8n \ --exclude-table-data=execution_data \ --exclude-table-data=execution_entity \ --exclude-table-data=execution_metadata \ > postgres/n8n.sql
3

Docker Volume Archive

n8n_storage와 qdrant_storage 볼륨을 tar.gz로 압축합니다. 커뮤니티 노드, 벡터 DB 데이터가 포함됩니다.

$ docker run --rm \ -v n8n_storage:/data:ro \ -v ./volumes:/backup \ alpine tar czf /backup/n8n_storage.tar.gz -C /data . $ docker run --rm \ -v qdrant_storage:/data:ro \ -v ./volumes:/backup \ alpine tar czf /backup/qdrant_storage.tar.gz -C /data .
4

Project Files Copy

docker-compose.yml, Dockerfile, .env 등 프로젝트 설정 파일과 기존 n8n/backup 디렉토리를 복사합니다.

# 복사되는 파일: docker-compose.yml # 서비스 구성 docker-compose_origin.yml Dockerfile # 커스텀 이미지 .env # 환경변수 (암호화 키 포함!) n8n/backup/ # 기존 import용 JSON
5

Metadata & ZIP

백업 정보(DB 통계, 컨테이너 상태, 커뮤니티 노드 목록)를 기록하고 전체를 하나의 ZIP 파일로 압축합니다.

# backup_info.txt에 기록되는 정보: - 백업 일시 및 호스트명 - Docker 이미지/컨테이너 상태 - DB 통계 (워크플로우, 크리덴셜, 사용자, 실행기록) - 커뮤니티 노드 패키지 목록 - 백업 파일 구조
File Structure
백업 파일 구조
백업 ZIP 파일을 압축 해제하면 아래와 같은 구조가 나타납니다.
backup_20260219_162358/ ├── backup_info.txt 메타데이터 & 복원 가이드 ├── postgres/ │ └── n8n.sql PostgreSQL 전체 덤프 ├── n8n_export/ │ ├── workflows/ 653개 JSON 파일 │ └── credentials/ 478개 JSON 파일 ├── volumes/ │ ├── n8n_storage.tar.gz 커뮤니티 노드 포함 │ └── qdrant_storage.tar.gz 벡터 DB 데이터 └── project/ ├── .env N8N_ENCRYPTION_KEY 포함! ├── docker-compose.yml ├── docker-compose_origin.yml ├── Dockerfile └── n8n/ ├── workflows/ └── credentials/
⚠

N8N_ENCRYPTION_KEY는 반드시 보관하세요

이 키가 없으면 크리덴셜을 복호화할 수 없습니다. .env 파일에 포함되어 있으며, 백업 ZIP 안의 project/.env에서 확인할 수 있습니다. 이 키를 분실하면 모든 API 키, OAuth 토큰 등을 다시 설정해야 합니다.

Restore Process
복원 7단계
restore.sh ./backup_20260219_162358 한 줄이면 충분합니다. 로컬 환경에서 복원할 때는 도메인을 localhost로 변경하세요.
1

백업 유효성 검증

n8n.sql, .env, docker-compose.yml, Dockerfile, backup_info.txt 존재 여부 확인

→
2

프로젝트 파일 복원

docker-compose.yml, Dockerfile, .env를 작업 디렉토리로 복사 (기존 .env는 자동 백업)

→
3

Docker 볼륨 복원

n8n_storage, qdrant_storage를 tar.gz에서 복원, postgres_storage는 빈 볼륨 생성

→
4

PostgreSQL 기동

커스텀 이미지 빌드 후 postgres만 먼저 기동, health check로 준비 완료 대기

→
5

DB 덤프 복원

기존 DB를 DROP → CREATE → psql로 SQL 덤프 파일 임포트

→
6

전체 서비스 기동

docker compose up -d — n8n, qdrant 등 나머지 서비스 시작, n8n health check 대기

→
7

복원 결과 검증

워크플로우, 크리덴셜, 사용자 수를 DB에서 조회하여 원본과 비교 검증

Claude Code Prompts
AI에게 이렇게 시키세요
이 백업 시스템을 처음부터 만들기 위해 Claude Code에 입력한 프롬프트 예시입니다. 순서대로 따라하면 동일한 결과를 얻을 수 있습니다.
1
현황 파악
필수
내 서버에서 docker compose로 n8n 돌리고 있어. 현재 구성 파악해줘
Claude가 하는 일: docker-compose.yml, .env, Dockerfile, 컨테이너 상태, DB 통계(워크플로우/크리덴셜/사용자 수) 등 서버 전체 현황을 분석합니다.
2
백업 스크립트 생성
필수
이 n8n 서버 전체를 백업하는 쉘 스크립트 만들어줘. 서비스 중단 없이 실행 가능해야 하고, 워크플로우/크리덴셜 JSON export, PostgreSQL 덤프, Docker 볼륨 tar, 프로젝트 파일까지 다 포함해줘. 실행기록 제외 옵션도 넣어줘
Claude가 하는 일: 5단계 backup.sh 스크립트를 생성합니다. n8n CLI export, pg_dump, 볼륨 아카이브, 프로젝트 파일 복사, 메타데이터 기록까지 자동화합니다.
3
복원 스크립트 생성
필수
백업한 걸 다른 서버에서 복원하는 스크립트도 만들어줘. 유효성 검증하고, DB 복원하고, 볼륨 복원하고, 서비스 기동까지 자동으로
Claude가 하는 일: 7단계 restore.sh 스크립트를 생성합니다. 백업 유효성 검증 → 파일 복원 → 볼륨 복원 → DB 복원 → 서비스 기동 → 결과 검증까지 한 번에 처리합니다.
4
백업 실행
상황에 따라
백업 스크립트 실행해줘. 실행기록은 빼고
Claude가 하는 일: ./backup.sh --no-executions를 실행하고 결과를 확인합니다. 직접 서버에서 실행해도 됩니다.
5
로컬 복원
상황에 따라
이 백업 파일 내 맥에서 Docker Desktop으로 복원해줘
Claude가 하는 일: 도메인을 localhost로 변경하고, restore.sh를 실행하여 로컬에서 동일한 환경을 구축합니다. 컨테이너명 불일치 등 환경 차이도 자동으로 처리합니다.
All-in-One Prompt

한 번에 다 시키고 싶다면

내 서버에서 docker compose로 n8n 돌리고 있어. 전체 백업 스크립트(서비스 중단 없이, 실행기록 제외 옵션 포함)랑 복원 스크립트 만들어줘. 백업 대상: n8n CLI export, PostgreSQL 덤프, Docker 볼륨, 프로젝트 파일(.env 포함). 복원은 다른 서버에서도 가능하게.
이 프롬프트 하나로 backup.sh + restore.sh 두 스크립트가 모두 생성됩니다.
